# Moodo - 気分優先タスク管理アプリ 要件定義書

## 概要

仕事のタスクに「作業見積もり・めんどくさレベル・期限・優先度」を設定し、「今の気分」をもとにAIがおすすめスコアを算出して表示するWebアプリ。ただのTODOリストではなく、そのときの気分を踏まえてタスクを選べるようにすることで、消化作業にならず自分の気分を優先した人間らしい働き方を支援する。

**スコープ**
- やること: タスク管理(CRUD) / 気分の入力 / AIによるおすすめスコアの算出と表示
- やらないこと: カレンダー連携 / チーム共有 / 繰り返しタスク / Claude APIのコード組み込み

## 背景・課題

- 既存のタスク管理ツール(Todoistなど)はタスクを羅列するだけで、使いこなしている実感がなかった。タイトルだけ並んでいてもメモ帳と変わらない
- 期限順にただこなすだけでは消化作業になり、仕事が楽しくなくなる
- タスクの難易度・期限・優先度などの属性を見える化し、そのときの気分を踏まえて「自分で選べる」状態にしたい
- 根本の動機: 仕事を楽しくしたい。これがコンセプト「人間らしく働く」に直結している

## 機能要件

### タスク管理

- タスクのCRUD（追加・編集・削除）。追加/編集はモーダルで行う
- タスク属性:
  - タイトル（テキスト）
  - 作業見積もり（極小 / 小 / 中 / 大 / 特大、フィボナッチポイント: 極小=1 / 小=2 / 中=5 / 大=8 / 特大=13）
  - めんどくさレベル（チョロ / まあまあ / 重い）
  - 期限（日付、一覧では相対表示・入力は MMDD 4桁）
  - 優先度（低 / 普通 / 高）
  - 詳細（テキスト、任意。モーダル内のみ表示・編集可能）
- タスク追加・編集モーダルで Ctrl+Enter によるキーボード送信に対応
- タスク削除時はブラウザ標準の confirm() ダイアログで誤削除を防止
- 画面の空きエリアをダブルクリックしてタスク追加モーダルを開ける（Ctrl+Enter も可）
- Moodoフラグ（今日やるフラグ）: タスクにON/OFFで設定できるフラグ
  - モーダルヘッダーの「☆ Moodo / ★ Moodo」ボタンでトグル操作
  - フラグONのタスクはタイトル前に★バッジを表示し、カード背景を黄色系に変更
  - ソートキーには含めない（スコア・期限・優先度のみでソート）
  - タスクカード上では直接操作できず、モーダル経由のみ

### 気分の入力

- ヘッダーに常時表示される入力欄
- 気分: セレクト（良い / 普通 / 微妙 / 悪い）
- 前回値を引き継ぎ、いつでも変更可能

### AI評価・おすすめスコア表示

- AI評価は手動フローで運用する（Claude APIのコード組み込みなし）
- 評価フロー:
  1. ユーザーが Claude Code に「タスク評価して」などと依頼
  2. Claude Code が `score-prompt.md` の手順に従い `tasks.json` と `mood.json` を読んでスコアを算出
  3. `tasks.json` の score フィールドを直接更新する
- AI評価前のタスクはスコアなしとして扱う
- おすすめスコア(0〜100%)を各タスクに表示
- スコアに応じて色分け:
  - 高（70%以上）: 緑
  - 中（40〜69%）: オレンジ
  - 低（39%以下）: グレー
- 色の閾値は後で調整できる構成にする

### タスク一覧

- スコア降順 → 期限昇順 → 優先度降順でソート
- 3カラムグリッド表示
- 各タスクに属性をチップ形式で表示（期限・作業見積もり・めんどくさレベル・優先度）
- 期限が今日かつ優先度が高のタスクは緊急表示（🔥バッジ）
- ヘッダーに今日期限・明日期限・今日完了・今週のポイントバッジを表示
  - 今日期限/明日期限: クリックで該当タスクに絞り込み（排他切り替え）
  - 今日完了: クリックで今日完了タスク一覧に切り替え
  - 今週: クリックでベロシティ画面へ遷移

### ベロシティ計測

- 週次ベロシティ画面（`/velocity`）: 過去12週の完了ポイントを折れ線グラフで表示
  - 今後3週分の予定ポイント（未完了タスクの期限ベース）を破線で表示
  - グラフの週をクリックして日別画面へ遷移
- 日別ベロシティ画面（`/velocity/week`）: 1週間の日別完了ポイントを折れ線グラフで表示
  - 今日以降の予定ポイントを破線で表示
  - 前週/次週ナビゲーション

### ピックアップエリア

- 画面下部に独立したセクションとして固定表示する
- 以下の2グループをページング（4.5秒ごとに自動切替）で表示する:
  - 今日期限: 期限が今日のタスクを最大3件
  - 積みタスク: 作成から7日以上経過したタスクを古い順に最大3件
- 全スロット表示後は30秒間非表示になり、その後再表示する
- 将来的には時間帯・期限・状況に応じて表示条件を切り替える予定
- ピックアップエリアに表示されたタスクはメインリストにも表示する（重複表示）

### データ保存

- `tasks.json` ファイルをバックエンドが読み書きする
- localStorageは使用しない

### UI/UX

- モーダルのクローズ操作（キャンセル・背景クリック・保存完了・Moodoボタン）は0.25秒フェードアウトアニメーションで閉じる
- ピックアップエリア（z-index: 100）よりモーダル（z-index: 200）が上のレイヤーに表示される

## 非機能要件

- 特になし（個人利用のため）
