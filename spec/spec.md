# Moodo - 気分優先タスク管理アプリ 要件定義書

## 概要

仕事のタスクに「作業見積もり・めんどくさレベル・期限・優先度」を設定し、「今の気分」をもとにAIがおすすめスコアを算出して表示するWebアプリ。ただのTODOリストではなく、そのときの気分を踏まえてタスクを選べるようにすることで、消化作業にならず自分の気分を優先した人間らしい働き方を支援する。

**スコープ**
- やること: タスク管理(CRUD) / 気分の入力 / AIによるおすすめスコアの算出と表示
- やらないこと: カレンダー連携 / チーム共有 / 繰り返しタスク / Claude APIのコード組み込み

## 背景・課題

- 既存のタスク管理ツール(Todoistなど)はタスクを羅列するだけで、使いこなしている実感がなかった。タイトルだけ並んでいてもメモ帳と変わらない
- 期限順にただこなすだけでは消化作業になり、仕事が楽しくなくなる
- タスクの難易度・期限・優先度などの属性を見える化し、そのときの気分を踏まえて「自分で選べる」状態にしたい
- 根本の動機: 仕事を楽しくしたい。これがコンセプト「人間らしく働く」に直結している

## 機能要件

### タスク管理

- タスクのCRUD（追加・編集・削除）。追加/編集はモーダルで行う
- タスク属性:
  - タイトル（テキスト）
  - 作業見積もり（小 / 中 / 大）
  - めんどくさレベル（チョロ / まあまあ / 重い）
  - 期限（日付、一覧では相対表示・入力は MMDD 4桁）
  - 優先度（低 / 普通 / 高）
  - 詳細（テキスト、任意。モーダル内のみ表示・編集可能）
- タスク追加・編集モーダルで Ctrl+Enter によるキーボード送信に対応
- タスク削除時はブラウザ標準の confirm() ダイアログで誤削除を防止
- 画面の空きエリアをダブルクリックしてタスク追加モーダルを開ける（Ctrl+Enter も可）

### 気分の入力

- ヘッダーに常時表示される入力欄
- 気分: セレクト（良い / 普通 / 微妙 / 悪い）
- 前回値を引き継ぎ、いつでも変更可能

### AI評価・おすすめスコア表示

- AI評価は手動フローで運用する（Claude APIのコード組み込みなし）
- 評価フロー:
  1. ユーザーが Claude Code に「タスク評価して」などと依頼
  2. Claude Code が `score-prompt.md` の手順に従い `tasks.json` と `mood.json` を読んでスコアを算出
  3. `tasks.json` の score フィールドを直接更新する
- AI評価前のタスクはスコアなしとして扱う
- おすすめスコア(0〜100%)を各タスクに表示
- スコアに応じて色分け:
  - 高（70%以上）: 緑
  - 中（40〜69%）: オレンジ
  - 低（39%以下）: グレー
- 色の閾値は後で調整できる構成にする

### タスク一覧

- スコア降順 → 期限昇順 → 優先度降順でソート
- 3カラムグリッド表示
- 各タスクに属性をチップ形式で表示（期限・作業見積もり・めんどくさレベル・優先度）
- 期限が今日かつ優先度が高のタスクは緊急表示（🔥バッジ）

### データ保存

- `tasks.json` ファイルをバックエンドが読み書きする
- localStorageは使用しない

## 非機能要件

- 特になし（個人利用のため）
