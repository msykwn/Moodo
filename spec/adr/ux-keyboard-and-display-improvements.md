# UX改善（キーボード操作・表示・入力形式の見直し）

## 概要

タスク入力・一覧表示のUXを複数まとめて改善した。
キーボードだけで完結する操作性の向上・入力形式の段階制への変更・一覧表示のわかりやすさ改善が目的。

**スコープ**
- やること: Ctrl+Enter でモーダルを開く / 作業見積もりを段階制に変更 / トグルボタン方式への統一 / キーボードナビゲーション / 期限の相対表示 / 今日締切バッジ / ソートキー追加 / autoFocus
- やらないこと: 既存データの自動移行処理（リリース前のため不要）

## 背景・課題

- **Ctrl+Enter**: タスク保存のショートカットだったが、モーダルを開く操作にキーボードショートカットがなかった
- **作業見積もり**: 分単位の数値入力は直感的でなく、厳密な時間よりざっくりした規模感で登録したい
- **プルダウン**: 見積もり・めんどくさレベル・重要度がすべてセレクトボックスで、選択状態が一目でわからない
- **キーボードナビゲーション**: タブキーでのフィールド遷移がスムーズでなく、隠し input やカレンダーボタンに止まる
- **期限表示**: `MM/DD` の日付表示より「今日」「明日」のほうが直感的
- **緊急タスクの視認性**: 期限が今日・重要度が高いタスクが一覧で目立たない
- **ソート**: スコア・期限が同一の場合の順序が不定だった

## 採用した変更と理由

### Ctrl+Enter でモーダルを開く

- **変更前**: Ctrl+Enter でフォーム送信（TaskModal 内のグローバルリスナー）
- **変更後**: Ctrl+Enter でタスク追加モーダルを開く（App.tsx のグローバルリスナー、モーダルが閉じているときのみ発火）
- **理由**: モーダルを開く操作にこそキーボードショートカットが有効。保存は Enter（submit）で完結する

### 作業見積もりを段階制（EstimateSize）に変更

- **変更前**: `estimate_minutes: int`（分単位の数値）
- **変更後**: `estimate_size: "極小" | "小" | "中" | "大" | "特大"`
- **理由**: タスクの作業量は厳密な時間よりも規模感で把握するほうが自然。5段階で十分な粒度を確保できる
- **影響**: バックエンドの Pydantic モデル・フロントの型・既存 tasks.json を更新。score-prompt.md のスコアリング説明も合わせて更新

### トグルボタン方式への統一

- 見積もり・めんどくさレベル・重要度の3フィールドをセレクトボックスからトグルボタングループに変更
- 選択中の値が青くハイライトされ、現在の設定が一目でわかる

### キーボードナビゲーション（ロービング tabIndex パターン）

- トグルグループに `role="radiogroup"` を設定し、選択中のボタンだけ `tabIndex=0`（他は `-1`）
- ←/→ または ↑/↓ キーでグループ内を移動し、値も同時に変更
- Tab でグループ外に抜ける
- 📅カレンダーボタン・隠し date input に `tabIndex={-1}` を設定し、Tab の止まり場所を排除
- タイトル input に `autoFocus` を設定し、モーダルを開いた瞬間に入力できる

### 期限の相対表示

- **変更前**: `MM/DD` 形式
- **変更後**: 今日 / 明日 / 明後日 / それ以降は `MM/DD`
- **理由**: 近い期限は相対表現のほうが緊急度が直感的に伝わる

### 今日締切バッジ（🔥）

- 期限が今日かつ重要度が「高」のタスクに 🔥 バッジを表示
- カード背景・ボーダーも薄赤にして視覚的に目立たせる

### ソート第三キーに重要度を追加

- **変更前**: スコア降順 → 期限昇順
- **変更後**: スコア降順 → 期限昇順 → 重要度降順（高 > 中 > 低）
- **理由**: スコア・期限が同一のタスクの順序を重要度で安定させる

## 状態

実装完了（2026-06-14）

## この決定によって生じる影響

**良い影響**
- Ctrl+Enter でモーダルを開けるため、マウス操作なしでタスク追加が完結する
- 規模感ベースの入力で登録がより直感的になる
- トグルボタンにより現在の選択値が一目でわかる
- キーボードのみでモーダルフォームを操作できる
- 今日締切の重要タスクが一覧で即座に目立つ

**悪い影響・トレードオフ**
- `estimate_minutes` → `estimate_size` の変更により既存 tasks.json の手動更新が必要（リリース前のため許容）
- スコアリングプロンプト（score-prompt.md）の `estimate_minutes` に関する記述を更新する必要があった
