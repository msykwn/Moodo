# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 要件定義書・設計書

- `spec/features/moodo-app.md` が現在の個別要件ファイル。機能要件・仕様の詳細はこちらを参照すること。
- 開発完了後、ユーザーの明示的な指示があったタイミングで `create-pull-request` スキル(`/create-pull-request`)を使い、個別要件ファイルの内容を `spec/requirements.md` にマージし、マージ済みファイルを `spec/adr/` へ ADR として移動したうえで GitHub に Pull Request を作成する。

## プロジェクト概要

仕事のタスクに「作業見積もり・めんどくさレベル・期限・重要度」を設定し、「今の気分」をもとにAIがおすすめスコアを算出して表示するWebアプリ。
ただのTODOリストではなく、自分の気分を優先した人間らしい働き方を支援することが目的。

## 技術スタック

- **フロントエンド**: React (Vite) + TypeScript
- **バックエンド**: FastAPI (Python) + uv
- **データ保存**: `tasks.json` ファイル（FastAPIが読み書き）
- **起動**: `npm run dev`（フロント）+ `uvicorn`（バック）の2コマンド

## AI評価の仕組み

Claude Code がタスクファイルを直接読み書きすることでスコアを更新する。エクスポート/インポートUIは存在しない。

### タスクの評価・整理をお願いする場合

ユーザーから「タスク整理して」「タスク評価して」「スコア更新して」「何やろうか」「何やったらよい」「今日何する」のような依頼が来たら、
`score-prompt.md`（プロジェクトルート）の内容を読み込み、そこに書かれた手順に従って実行すること。

具体的には以下を行う:
1. `backend/tasks.json` を読む（タスク一覧・各タスクの属性）
2. `backend/mood.json` を読む（現在の気分）
3. スコアリングの考え方に従って各タスクの `score` を 0〜100 の整数で算出する
4. `tasks.json` の `score` フィールドだけを更新して上書き保存する（他のフィールドは変更しない）

## AIスコアリングの既知の遅さについて

`/tasks/score/run` エンドポイントは `claude -p` をサブプロセスとして起動し、ClaudeにJSONファイルの読み書きをさせている。このため以下の理由で実行に数十秒かかる。

- Claude CLIのプロセス起動コスト（Node.jsランタイムの立ち上げ）が毎回発生する
- Claudeがエージェントループを走らせ、ファイルRead→スコア計算→ファイルWriteと複数ステップを踏む

改善する場合は、Python側でファイルを読んでプロンプトに埋め込み `--no-tools` で出力JSONだけ返させる方式、またはAnthropic APIを直接叩く方式が有効。
